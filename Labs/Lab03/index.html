Subfunctions

Last week we wrote function files that included only a single function. As functions become increasingly encumbered with loops, conditionals, print statements, plotting, and complicated computations, it can be beneficial in several ways to divide the main function into subfunctions. In computer science this process is called abstraction, and it basically means to hide fine details from view in order to focus on larger ideas. Below is a subfunction template for ENS 1300: 

% Purpose: mainFunc() is a template for ENS1300
% Inputs: input - input is just a placeholder
% Output: output - output is just a placeholder
% Usage: functionTemp()
% Subfunctions: 
    % subFunc1: it says hello 
    % subFunc2: it says hello
% Notes: ...
function mainOutput = mainFunc(mainInput)
    subFunc1();
    subFunc2();
end

function output = subFunc1(input)
% Inputs: input - input is just a placeholder
% Output: output - output is just a placeholder
    fprintf('subFunc1: Hello, I''m the first subfunction!\n')
end

function output = subFunc2(input)
% Inputs: input - input is just a placeholder
% Output: output - output is just a placeholder
    fprintf('subFunc2: Hello, I''m the second subfunction!\n')
end

Note that the input and output definitions are only placeholders. Because no inputs are ever used in the template subfunctions, the following code will behave in the exact same way.

function mainFunc()
    subFunc1();
    subFunc2();
end

function subFunc1()
    fprintf('subFunc1: Hello, I''m the first subfunction!\n')
end

function subFunc2()
    fprintf('subFunc2: Hello, I''m the second subfunction!\n')
end

Important things to consider when using subfunctions:

  • Everything in the file must be a function.
  • The first function is the 'main' or 'primary' function and is the one that is called when the file is run.
  • None of the other functions (subfunctions) can be referenced from outside the file.
  • Each subfunction begins with its own function definition line. The functions immediately follow each other. The various subfunctions can occur in any order, as long as the primary function appears first.
  • Each function must be terminated by an end statement.

There are many ways to organize code by using subfunctions. In this course, we will primarily use main functions to call subfunctions only. In other words, the main function will show the big idea and the subfunctions will contain the details. Through appropriate function naming, the programmers intent can be made clear just by reading the main function. For example:

function myMainFunc(fileName)
    data = readInput(fileName);
    results = performCalculations(data);
    generatePlots(results);
    writeOutputFile(results);
end

function importedData = readInput(file)
    % Code that imports data from some file or web source
end

function calcResults = performCalculations(someData)
    % Code that performs some calculation on the imported data
end

function generatePlots(plotData)
    % Code that creates plots using the calculation results
end

Data Types

MATLAB supports many different data types. The type, or class, of a variable designates what data it can store, how much memory it will require, and what operations can be performed on it. Some of the most common MATLAB data types are listed below in bold font.

  • Numerical
    • Floating Point
      • single - uses 32 bits of memory
      • double - uses 64 bits of memory (default)
    • Integers
      • uint8, int8 - unsigned and signed 8 bit integers
      • uint16, int16 - unsigned and signed 16 bit integers
      • uint32, int32 - unsigned and signed 32 bit integers
      • uint64, int64 - unsigned and signed 64 bit integers
  • Characters and Strings
    • char - character arrays (single quotes)
    • string - strings and string arrays (double quotes)
  • Containers
    • struct - structure
    • cell - cell arrays
  • Function Handles
    • function_handle - anonymous functions and function references
  • Boolean
    • logical - (1=true, 0=false)

Data Import/Export

There are several different built-in methods in MATLAB for the import/export of data and many more supported file format types. For now, we'll only look at 3 import/export format types.

  1. Comma Separated Value (.csv)
  2. JavaScript Object Notation (.json)
  3. MATLAB workspace file (.mat)

Functions for downloading files from URL:

  • websave (MATLAB only)
  • urlwrite (MATLAB and Octave)

Import/Export CSV format

The comma separated value format is of widespread use for scientific applications. It is especially suited for recording and/or moving tabular data. The following is an example of a plain text CSV file.

Header 1,Header 2,Header 3,Header 4,Header 5
1,4,7,10,13
3,5,8,11,14
3,6,9,12,15

And it would look like this if opened with spreadsheet software.

Header 1 Header 2 Header 3 Header 4 Header 5
1 4 7 10 13
3 5 8 11 14
3 6 9 12 15

Functions to import data from local CSV files.

  • dlmread
  • csvread
  • importdata

Functions to export in CSV format.

  • dlmwrite
  • csvwrite

Example: Import CSV data using websave() and dlmread()

Import data from a CSV file, CSVexample.csv, located at the following URL: https://c-a-j.github.io/ENS1300/CSVexample.csv

URL = 'https://c-a-j.github.io/ENS1300/CSVexample.csv'; 
fn = 'example.csv'; % filename for local csv file
websave(fn, URL); % save the file
data = dlmread(fn, ',', 1, 0); % read the data to a matrix

Example: import CSV data using websave() and csvread()

csvread() is almost the same as dlmread(), but you don't have to specify the delimiter.

URL = 'https://c-a-j.github.io/ENS1300/CSVexample.csv'; 
fn = 'example.csv'; % filename for local csv file
websave(fn, URL); % save the file
data = csvread(fn, 1, 0); % read the data to a matrix

Example: Import CSV data using websave() and importdata()

importdata() will attempt to split the table into its text and numerical parts into a structure. For tables with complicated headers, specific options may need to be specified.

URL = 'https://c-a-j.github.io/ENS1300/CSVexample.csv'; 
fn = 'example.csv'; % filename for local csv file
websave(fn, URL); % save the file
data = importdata(fn); % read the data to a structure

M = randi(100,5,5); % generate random matrix of integers
fn = 'randomMatrix.csv'; % csv filename
dlmwrite(fn, M, ','); % write the matrix, M, to csv file

M = randi(100,5,5); % generate random matrix of integers
fn = 'randomMatrix.csv'; % csv filename
csvwrite(fn, M); % write the matrix, M, to csv file

Import/Export JSON format

JavaScript Object Notation JSON is the most widely-used file format for REST APIs. The details of REST APIs are a bit beyond the scope of this course, but you can think of them simply as a set of rules between web-based systems that allow them to share information. For now, we'll ignore REST APIs and focus on what the data might look like in MATLAB once a request has been made.

The JSON file containing Joe Schmo's information can be accessed at the following URL: https://c-a-j.github.io/ENS1300/JSONexample.json. The examples below show two different ways to import the data.

Example: Import from JSON

Using jsondecode():

URL = 'https://c-a-j.github.io/ENS1300/JSONexample.json';
fn = 'example.json'; % json filename
websave(fn, URL); % save the json file
file = fopen(fn, 'r'); % open the json file
txt = fread(file); % read in the json file
data = jsondecode(sprintf('%s',txt)); % interpret the json file

And using webread()

URL = 'https://c-a-j.github.io/ENS1300/JSONexample.json';
data = webread(URL); % get data directly from url

Both methods result in a structure called data that contains all Joe Schmo's information. Try accessing Joe's quiz scores using data.JoeSchmo.scores.quizzes or other programming languages he knows with data.JoeSchmo.other_languages.

The following examples import Joe Schmo's information, add a new field, and then save the MATLAB struct as a new JSON file.

Import/Export MATLAB Workspace (.mat)

In most cases it is best to import/export using a standard format such as CSV or JSON. However, if MATLAB (or Octave) is guaranteed to be the programming language used for some analysis/program, then utilizing workspace files would be suitable.

The following example imports data from a JSON file then saves the resulting structure in a MATLAB workspace file.

Example: Exporting single variable to a workspace file (.mat)

URL = 'https://c-a-j.github.io/ENS1300/JSONexample.json';
data = webread(URL); % get data directly from url

fn = 'myWorkspaceFile';
save(fn, 'data')

Example: Exporting multiple variables to a workspace file (.mat)

URL = 'https://c-a-j.github.io/ENS1300/JSONexample.json';
data = webread(URL); % get data directly from url
x = 0:100;
y = linspace(0,45, length(x));
z = 0:0.1:100;

fn = 'myWorkspaceFile';
save(fn, 'data', 'x', 'y')

Example: Exporting all variables to a workspace file (.mat)

URL = 'https://c-a-j.github.io/ENS1300/JSONexample.json';
data = webread(URL); % get data directly from url
x = 0:100;
y = linspace(0,45, length(x));
z = 0:0.1:100;

fn = 'myWorkspaceFile';
save(fn)

The previous three examples all result in a new file named myWorkspaceFile.mat. Importing the saved data from another script or function would look like this:

load('myWorkspaceFile');

Using Structures as Function Input/Output

One of the main decisions to make when using structures as the input/output of user-created functions is whether or not to retain the original values.

Example: Excluding original structure values from output

function newStruct = mainFunc()
    %%% Initialize
    orgStruct.v = 6;
    orgStruct.w = 7;
   
    %%% Execute 
    newStruct = subFunc(orgStruct);
end

function outStruct = subFunc(inStruct)
    outStruct.x = inStruct.v.*inStruct.w;
    outStruct.y = inStruct.v./inStruct.w;
    outStruct.z = inStruct.v + inStruct.w;
end

When the code above is executed, only the results x, y, and z are contained in the resulting structure.

Example: Retaining original structure values (1)











 





function newStruct = mainFunc()
    %%% Initialize
    orgStruct.v = 6;
    orgStruct.w = 7;
   
    %%% Execute 
    newStruct = subFunc(orgStruct);
end

function outStruct = subFunc(inStruct)
    outStruct = inStruct; % insert orginal values into the new struct
    outStruct.x = inStruct.v.*inStruct.w; % add x
    outStruct.y = inStruct.v./inStruct.w; % add y
    outStruct.z = inStruct.v + inStruct.w; % add z
end

When the code above is executed, all values v, w, x, y, and z are contained in the resulting structure. This code is also equivalent to the following:







 



 





function myStruct = mainFunc()
    %%% Initialize
    myStruct.v = 6;
    myStruct.w = 7;
   
    %%% Execute 
    myStruct = subFunc(myStruct);
end

function outStruct = subFunc(inStruct)
    outStruct = inStruct; % insert orginal values into the new struct
    outStruct.x = inStruct.v.*inStruct.w; % add x
    outStruct.y = inStruct.v./inStruct.w; % add y
    outStruct.z = inStruct.v + inStruct.w; % add z
end

Another way to retain values of a structure is to define the output of the function to be the same as the input.










 





function newStruct = mainFunc()
    %%% Initialize
    orgStruct.v = 6;
    orgStruct.w = 7;
   
    %%% Execute 
    newStruct = subFunc(orgStruct);
end

function myStruct = subFunc(myStruct)
    myStruct.x = myStruct.v.*myStruct.w; % add x
    myStruct.y = myStruct.v./myStruct.w; % add y
    myStruct.z = myStruct.v + myStruct.w; % add z
end

When the code above is executed, all values v, w, x, y, and z are contained in the resulting structure. This code is also equivalent to the following:







 


 





function myStruct = mainFunc()
    %%% Initialize
    myStruct.v = 6;
    myStruct.w = 7;
   
    %%% Execute 
    myStruct = subFunc(myStruct);
end

function myStruct = subFunc(myStruct)
    myStruct.x = myStruct.v.*myStruct.w; % add x
    myStruct.y = myStruct.v./myStruct.w; % add y
    myStruct.z = myStruct.v + myStruct.w; % add z
end

You should start to see by now that there are many different ways to accomplish the same thing in programming.
