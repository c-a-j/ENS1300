Example: Plotting in ENS 1300

You will assign variable names to most figure windows/objects for this class. This allows you to refer to the figure later in the script or function to save it as an image file and ensures that separate windows are opened rather than overwriting with subsequent plot commands.

The following example is similiar to Problem 4, Lab 2.

x = -5:0.1:5;
y1 = x.^2;
y2 = x.^3;

% first figure open and plot
fig1 = figure(1);
plot(x, y1, 'b-', x, y2, 'r-'); 

% second figure open and plot
fig2 = figure(2);
subplot(2, 1, 1)
plot(x, y1, '-b');
subplot(2, 1, 2)
plot(x, y2, '-r');

And saving the plots as images might look like this:

x = -5:0.1:5;
y1 = x.^2;
y2 = x.^3;

% first figure open, plot, and save
fig1 = figure(1);
plot(x, y1, 'b-', x, y2, 'r-'); 
saveas(fig1,'firstFigure','png')

% second figure open, plot, and save
fig2 = figure(2);
subplot(2, 1, 1)
plot(x, y1, '-b');
subplot(2,1,2)
plot(x, y2, '-r');
saveas(fig1, 'secondFigure', 'png')

Try running the above code segment in a new script. Image files firstFigure.png and secondFigure.png should appear in your current directory. In programming, there are always many different ways to achieve the same thing. The previous code and the following are equivalent:

x = -5:0.1:5;
y1 = x.^2;
y2 = x.^3;

% first figure open, plot, and save
figure(1);
plot(x, y1, 'b-', x, y2, 'r-'); 
saveas(gcf,'firstFigure','png'); % gcf is a built-in variable

% second figure open, plot, and save
figure(2);
subplot(2, 1, 1)
plot(x, y1, '-b');
subplot(2,1,2)
plot(x, y2, '-r');
saveas(gcf, 'secondFigure', 'png'); % gcf = current figure handle
