About

I recently received a MS degree from UTK in Environmental Engineering. While there, I worked several years as a graduate teaching assistant in the Engineering Fundamentals Program, including a few semesters with the MATLAB course, EF 230.

Intro to ENS 1300

In this class you will practice solving problems through computer programming, and you'll discover the need to apply a methodical, detail oriented, tenacious approach. Programming is a skill that requires significant determination to develop, and you will often have to independently seek out resources to find solutions that fit your needs.

We will be using videos and lecture materials from UT's Engineering Fundamentals MATLAB course, EF 230. This website will serve simply to link you to the appropriate locations in the EF 230 website and to add details where there are any differences in the material covered.

NOTE: You are not expected to have significant programming experience when you start this class, and you are not expected you to be an expert programmer at the end of this class. The main goal of this course is to develop an understanding of programming fundamentals.

To be successful in this course, it is recommended to:

  1. Turn assignments in on time.
    • There will be a penalty applied to all late submissions.
  2. Come to class.
    • It's the best time to ask questions.
  3. Don't get behind.
    • The concepts and methods presented in this course take time to understand. Getting behind is sure to cause undue stress/aggrivation.
  4. Ask questions.
    • Always put forth a solid individual attempt, but feel free to ask questions in class, in office hours, or via email.
  5. Focus intensely on the basics!
    • Diligence in the beginning will pay off greatly. Quickly developing a firm understanding of the basics will likely make the rest of the course a breeze.
  6. Consider purchasing a MATLAB student license.
    • It will be convenient for you to have a local copy of MATLAB on your personal computer.

Accessing MATLAB on your computer

You can purchase a MATLAB student license here. It is $49 for MATLAB only and $99 for MATLAB plus many useful toolboxes. It is strongly recommended to purchase the $99 license if you decide to go this route. When compared to textbooks that may be upwards of $200, the student license is not a significant expense for course materials. For example, at UT you would pay $65 extra per engineering course, a $120 technology fee, and often a $100-$200 textbook will be required.

Another option is to install Octave on your personal computer for free. Most, if not all, of the code that we write for this course can be ran in Octave without any modifications.

A third option is to code in Python and use the Spyder IDE, which is free and is similar to MATLAB. Install it with the Anaconda Distribution. I will gladly accept Python code for all assignments and exams.

The MATLAB IDE

To make interfacing with a programming language a bit more palatable, many programmers prefer to use an IDE (Integrated Development Environment). When using one of the many open-source languages such as C++ or Python, the user would need to find, download, install, and configure the IDE manually. MATLAB, however, is proprietary and has a very nice built-in IDE.

Address Bar and Current Folder Window

The present working directory is always shown in the address bar and the contents of that directory are displayed in the current folder window. Getting a grip on the working directory is an important first step in learning how to program. It's the default location of new scripts and, unless specified otherwise, it is where MATLAB will look when attempting to find input files or user functions. For an in-depth explanation, go here.

The working directory above is /Users/c-a-j/Sites/ENS1300/blocks/ENS1300. Notice that you can also see the present working directory by typing pwd in the command window.

Command and Workspace Windows

For the purpose of this course, the command window can be considered a glorified calculator. It is mainly used to perform quick calculations or for testing code segments. Try defining a few variables and executing a calculation on them. When you're finished, try scrolling through the command history by using the up and down arrow keys.

While we can consider the command window as just an extremely powerful calculator, it is actually where most of the magic happens. It runs the MATLAB environment that interprets code inside m-files (.m) and is the main location for outputting results.

The workspace window simply provides information about the variables that are currently defined within the MATLAB environment. In the above screenshot, the variables x, y, and z have been explicitly defined in the command window and the variable ans, short for answer, is assigned the value from the first execution, 2+2, because a variable name was not specified.

Editor Window

The editor window is an advanced text editor designed specifically for the MATLAB language that will be used to create script and function files. This is where the majority of the work for this course will be completed. There is essentially no difference between executing a line of code in the command window and executing it in an m-file. The advantage of using scripts and functions is that large amounts of code can be executed rapidly and repetitively with ease.

ENS 1300 Code Organization

We'll use a specific programming pattern for this course. Strictly adhering to the patten will greatly increase the readability of your code, making it easier for me to grade and for you to revisit/understand later. The template that you should start with for all ENS 1300 scripts is shown below.

% Name:
% Section:
% Assignment:
% Description:
% Date: 
clear; close all; clc; format compact; format short;

%% Initialize

%% Read

%% Execute

%% Print

Copy and paste the contents in a new script or download the template directly here.

Initialize Section

This is where input from inside the program will be defined. For example, if you were working a problem involving the trajectory equation, the initialize section is where g, θ, v0, x0, and y0 would all be defined.

Read Section

This is where input from outside the program will be read in. Inputs in this course may come from the command window via the input() function or from a csv (comma separated value) text file via the csvread() function. More on this later.

Execute Section

All calculations go in this section.

Print Section

The print section will contain output commands of all types. Outputs for this course will usually be plots, command window messages, and maybe text files.

File Management

File management is extremely important in computer programming. In this course, we will be working with at least one new MATLAB file with each lab and occasionally additional input files. It will be convenient for you (and me) to adopt a consistent and meaningful file structure.

We will use a specific file structure for this course. The reason for this is not only for your own organization, but that your submissions will be evaluated partially by scripts. If your file system is not in the form the grading scripts expect, they won't be found. Follow the steps below to create the file structure:

Zipping/Compressing

For our purposes, zipping files is simply a means of packaging multiple files and directories together into a single file for assignment submissions. Instead of uploading individual files, you'll essentially be uploading a submission-time snapshot of the entire directory. If you'd like to know more about the file format, go here. Follow the steps below to create your first zip file:

  1. Go back to the previous section and verify that your ENS1300 directory has all the required components.
  2. Decide on which method is right for you.
    • Mac
    • Windows
    • or do it from inside MATLAB!!
  3. Move the script template file into the ENS1300/Templates folder.
    • If you skipped over creating the template, go back to ENS 1300 Code Organization.
  4. Create a zip file of your ENS1300 folder.
    • Name the zip file ENS1300_LastnameFirstname.zip and move it to ENS1300/Labs/Lab01.

TIP: How to do it from inside MATLAB

  • MATLAB has a built-in function called zip for zipping files and folders. The usage for the function is zip(zipfilename, foldername), where zipfilename and foldername can both be full file paths.
  1. If you're current working directory is ENS1300, type the following in the command window then press enter.
zip('../ENS1300-01', '.')
  • Note: The . means the current directory, and .. means one path level up in the directory addresses above.
  1. Use movefile to move the zip file to ENS1300/Labs/Lab01. The usage is movefile 'source' 'destination'
movefile '../ENS1300-01.zip' 'Labs/Lab01/ENS1300-01.zip'
